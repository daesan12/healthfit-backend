## 6. DB 설계 방향

### 선택 방식

JSONField 사용

예시

```python
body_parts = models.JSONField()
equipments = models.JSONField()
target_muscles = models.JSONField()
secondary_muscles = models.JSONField()
instructions = models.JSONField()
```

### 정규화를 하지 않은 이유

1. 운동 데이터 규모가 크지 않음 (415개)
2. 조회가 대부분이며 수정이 거의 발생하지 않음
3. 프로젝트 기간 내 핵심 기능 구현 우선
4. JSONField를 활용하면 개발 속도가 빠름
5. 추후 서비스 확장 시 정규화 가능

## 7. 최종 Exercise 모델

```python
from django.db import models

class Exercise(models.Model):

    exercise_id = models.CharField(
        max_length=20,
        unique=True
    )

    name = models.CharField(
        max_length=100
    )

    gif_url = models.URLField()

    body_parts = models.JSONField()

    equipments = models.JSONField()

    target_muscles = models.JSONField()

    secondary_muscles = models.JSONField()

    instructions = models.JSONField()

    def __str__(self):
        return self.name
```

### 비고

- PK(id)는 Django가 자동 생성
- exercise_id는 원본 ExerciseDB 식별자 보존용
- 향후 데이터 재수집 및 중복 방지에 활용 가능
