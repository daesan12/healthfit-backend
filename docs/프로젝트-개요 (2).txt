# 1. 서비스 개요

본 서비스는 사용자의 신체 정보, 운동 목표, 활동량, 식단 기록, 운동 기록을 기반으로 개인에게 적합한 식단과 운동 루틴을 추천하는 AI 기반 건강 관리 플랫폼이다.

사용자는 회원가입 후 마이프로필에 자신의 성별, 나이, 키, 몸무게, 체형, 활동량, 운동 경험, 운동 목표 등을 입력한다. 서비스는 해당 정보를 바탕으로 하루 권장 섭취 칼로리와 탄수화물·단백질·지방 비율을 계산하고, 저장된 음식 데이터와 운동 데이터를 활용하여 개인 맞춤형 식단과 운동 루틴을 제공한다.

또한 사용자는 자신의 식단과 운동 기록을 저장하고, AI로부터 오늘의 식단 평가와 운동 루틴 피드백을 받을 수 있다. 커뮤니티 기능을 통해 다른 사용자와 식단, 운동 루틴, 운동 기록을 공유할 수 있으며, 좋아요와 댓글을 통해 상호작용할 수 있다.

---

# 2. 서비스 핵심 목표

본 프로젝트의 핵심 목표는 단순한 음식 검색 또는 운동 목록 제공이 아니라, 사용자의 목표 달성을 돕는 개인 맞춤형 건강 관리 서비스를 구현하는 것이다.

주요 목표는 다음과 같다.

첫째, 사용자의 신체 정보와 목표를 기반으로 하루 권장 칼로리와 영양 비율을 계산한다.

둘째, 음식 데이터베이스를 활용하여 목표 칼로리와 탄단지 비율에 맞는 식단을 추천한다.

셋째, 사용자의 운동 경험과 목표를 기반으로 적절한 운동 루틴을 추천한다.

넷째, 사용자가 기록한 하루 식단을 분석하여 점수와 피드백을 제공한다.

다섯째, 사용자가 자신의 식단과 운동 루틴을 저장하고 커뮤니티에 공유할 수 있도록 한다.

---

# 3. 전체 사용자 흐름

1. 사용자는 회원가입 또는 로그인을 진행한다.
2. 마이프로필에서 성별, 나이, 키, 몸무게, 체형, 활동량, 운동 경험, 운동 목표를 입력한다.
3. 서비스는 프로필 정보를 기반으로 하루 권장 칼로리와 탄단지 권장 비율을 계산한다.
4. 사용자는 음식 검색 또는 직접 음식 추가 기능을 통해 식단 데이터를 구성한다.
5. AI 식단 추천 페이지에서 목표에 맞는 식단을 추천받고 저장할 수 있다.
6. 사용자는 아침, 점심, 저녁, 간식 단위로 하루 식단을 기록한다.
7. 오늘의 식단 평가 기능을 통해 총 섭취 칼로리, 탄단지 비율, 부족하거나 과다한 영양소 피드백을 확인한다.
8. AI 운동 추천 페이지에서 체형, 운동 경험, 운동 목표에 맞는 운동 루틴을 추천받는다.
9. 추천받은 운동 루틴을 수정하거나 저장하고, 실제 수행한 운동 기록을 남긴다.
10. 진행 현황 페이지에서 체중 변화, 섭취 칼로리, 운동 기록, 식단 점수 변화를 확인한다.
11. 사용자는 자신의 식단, 운동 루틴, 운동 기록을 커뮤니티에 공유하고 댓글과 좋아요로 소통한다.

# 4. 주요 기능

## 4.1 회원 및 마이프로필 기능

### 기능 설명

사용자는 회원가입과 로그인을 통해 개인 계정을 생성하고, 마이프로필에서 식단 및 운동 추천에 필요한 기본 정보를 입력한다.

마이프로필은 서비스의 모든 추천 기능의 기준 데이터가 된다. 따라서 단순 개인정보 화면이 아니라 식단 추천, 운동 추천, 식단 평가, 진행 현황 분석에 직접 연결되는 핵심 기능이다.

## 4.2 권장 칼로리 계산 기능

### 기능 설명

사용자의 성별, 나이, 키, 몸무게, 활동량, 운동 목표를 기반으로 하루 권장 섭취 칼로리를 계산한다.

이 기능은 AI 식단 추천의 기준값이 되며, 사용자가 하루 동안 섭취한 칼로리가 목표 대비 적절한지 평가하는 기준으로도 사용된다.

## 4.3 음식 데이터 관리 기능

### 기능 설명

서비스에서 식단 추천과 식단 기록에 활용할 음식 데이터를 관리하는 기능이다.

기본 음식 데이터는 DB에 저장하고, 사용자가 직접 새로운 음식을 추가할 수도 있다. 사용자가 직접 추가한 음식은 개인 식단 기록과 추천 식단 구성에 활용할 수 있다.

## 4.4 AI 식단 추천 기능

### 기능 설명

사용자의 마이프로필 정보와 권장 칼로리 계산 결과를 기반으로 하루 식단을 추천하는 기능이다.

AI는 사용자의 목표와 탄단지 비율을 고려하여 아침, 점심, 저녁, 간식으로 구성된 식단을 추천한다. 추천 결과에는 총 칼로리, 탄수화물, 단백질, 지방, 추천 이유가 포함된다.

## 4.5 식단 기록 기능

### 기능 설명

사용자가 하루 동안 섭취한 음식을 아침, 점심, 저녁, 간식 단위로 기록하는 기능이다.

식단 기록은 오늘의 식단 평가, 진행 현황 분석, AI 피드백의 기준 데이터로 사용된다.

## 4.6 오늘의 식단 평가 기능

### 기능 설명

사용자가 하루 동안 기록한 식단을 분석하여 점수와 피드백을 제공하는 기능이다.

오늘의 식단 평가는 사용자의 권장 칼로리와 실제 섭취 칼로리, 권장 탄단지 비율과 실제 탄단지 비율을 비교하여 제공한다.

## 4.7 운동 데이터 조회 기능

### 기능 설명

서비스에 등록된 운동 데이터를 조회하고, 운동별 상세 정보를 확인할 수 있는 기능이다.

운동 데이터는 AI 운동 추천과 운동 루틴 생성의 기준 데이터로 사용된다.

## 4.8 AI 운동 추천 기능

### 기능 설명

사용자의 체형, 운동 경험, 운동 가능 시간, 운동 목표를 기반으로 운동 루틴을 추천하는 기능이다.

AI는 사용자의 운동 경험을 고려하여 무리한 루틴이 아닌 현실적으로 수행 가능한 루틴을 추천한다.

## 4.9 운동 루틴 관리 기능

### 기능 설명

사용자는 AI가 추천한 운동 루틴을 저장하거나 직접 수정하여 자신만의 운동 루틴을 만들 수 있다.

## 4.10 운동 기록 저장 기능

### 기능 설명

사용자가 실제로 수행한 운동 기록을 저장하는 기능이다.

운동 기록은 진행 현황 분석과 커뮤니티 공유에 활용된다.

## 4.11 진행 현황 조회 기능

### 기능 설명

사용자의 식단 기록, 운동 기록, 신체 기록을 기반으로 기간별 변화를 확인하는 기능이다.

## 4.12 AI 질문 응답 기능

### 기능 설명

사용자가 식단, 운동, 영양 정보에 대해 자유롭게 질문하고 AI의 답변을 받을 수 있는 기능이다.

## 4.13 커뮤니티 기능

### 기능 설명

사용자는 자신의 식단, 운동 루틴, 운동 기록을 게시글 형태로 공유할 수 있다.

커뮤니티 기능은 서비스의 재미 요소와 지속 사용성을 높이는 기능이다. 사용자는 다른 사람의 루틴과 식단을 참고할 수 있고, 댓글과 좋아요를 통해 상호작용할 수 있다.

# 5. AI 활용 방식

본 프로젝트에서 AI는 단순한 챗봇이 아니라 사용자의 식단과 운동 데이터를 분석하고 설명하는 역할을 담당한다.

AI가 담당하는 주요 역할은 다음과 같다.

- 식단 추천 이유 생성
- 식단 평가 피드백 생성
- 부족하거나 과다한 영양소 설명
- 운동 루틴 추천 이유 생성
- 운동 경험에 따른 주의사항 제공
- 사용자의 질문에 대한 답변 제공

# 6. 추천 시스템 설계

## 6.1 식단 추천 기준

식단 추천은 다음 기준을 활용한다.

- 사용자 성별
- 사용자 나이
- 키
- 몸무게
- 활동량
- 운동 목표
- 하루 권장 칼로리
- 권장 탄단지 비율
- 음식별 칼로리
- 음식별 탄수화물
- 음식별 단백질
- 음식별 지방

## 6.2 운동 추천 기준

운동 추천은 다음 기준을 활용한다.

- 사용자 체형
- 운동 목표
- 운동 경험
- 운동 가능 시간
- 운동 빈도
- 운동 난이도
- 운동 부위
- 예상 소모 칼로리

## 6.3 추천 결과 저장

추천 결과는 단순히 화면에 출력하고 끝나는 것이 아니라 사용자가 다시 확인할 수 있도록 저장 가능해야 한다.

식단 추천 결과는 저장 식단으로 저장하고, 운동 추천 결과는 운동 루틴으로 저장한다.

음식 데이터는 식단 추천, 식단 기록, 식단 평가에 사용된다.

- 칼로리
- 탄수화물
- 단백질

- 난이도
- MET 값

- 수행 방법

사용자 기록 데이터는 진행 현황 분석에 사용된다.

주요 데이터는 다음과 같다.

- 신체 기록
- 식단 기록
- 운동 기록
- 식단 평가 기록
- 진행 현황 기록

# 8. 활용 예정 API

## 8.1 식품 영양 API

외부 API는 확장 기능으로 활용한다.

활용 가능한 API 예시는 다음과 같다.

- 공공데이터포털 식품영양성분 API
- Nutritionix API
- Edamam API

1차 구현에서는 직접 구축한 음식 DB를 우선 사용하고, 이후 외부 API를 연동하여 음식 데이터의 범위를 확장한다.

## 8.2 AI API

AI 기능 구현에는 다음 API 중 하나를 활용할 수 있다.

- OpenAI API
- Gemini API

AI API는 식단 추천, 식단 평가 피드백, 운동 추천, 질문 응답 기능에 사용한다.
